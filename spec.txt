Okay, this is meant to be a kind of small-scale fitness tracker that should run for just one person. TechStack is that this is running as a website. It has a password. The backend is running in Cloudflare worker, and the data is stored in a Cloudflare D1 database. And that's also how it's deployed. We need to figure out how to compile the frontend. It should use something like React or Svelte, doesn't really matter. And when you go to that website, then first you should log in with the password. It should remember locally in a cookie or something, via session cookie or something like this. Then it should give you access to the fitness data. What we want to track is training sessions, and each training session has a date, obviously, and then it has different exercises, which are performed with different numbers of repetitions. So, we want to track the fitness data. And then what we can show with nice looking charts, line charts in the best case, is how the weight and the repetitions and so on develop over time for the individual exercises across the trainings.

Business objects:
* Exercise (e.g. Latzug, Rudern, Schraegbank)
* Exercise category (Oberkoerper, Ganzkoerper, Unterkoerper)
* Training: Date, and for a number of exerices it has a single weight and a number of repetitions over a defined numver of sets (e.g. latzug had 12-10-10 repetitions which are 3 sets, with a weight of 70) most of the time there is just one weight per exercise and training, but sometimes it's per set, e.g. each Latzug set has a different weight
